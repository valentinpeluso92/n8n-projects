# Skill: Brainstorming Pro

## Cuándo usar este skill
- Cuando el usuario pida ideas, variantes, conceptos, hooks, nombres, formatos o enfoques.
- Cuando haya bloqueo creativo o demasiadas opciones y haga falta ordenar.
- Cuando el usuario necesite ideas “buenas para ejecutar”, no solo ocurrencias.

## Inputs necesarios (si faltan, pregunta primero)
1) **Objetivo exacto**: Qué se quiere conseguir.
2) **Público / contexto**: Para quién es y dónde se usa.
3) **Restricciones**: Tiempo, presupuesto, tono, formato, herramientas.
4) **Ejemplos**: Ejemplos de lo que SÍ y lo que NO (si el usuario tiene preferencias).

## Workflow
1) **Aclarar**: Aclara el encargo con 3–5 preguntas rápidas (solo si faltan datos).
2) **Generar**: Genera ideas en 4 tandas:
    - **A) 10 ideas rápidas**: Claras y ejecutables.
    - **B) 5 ideas “diferentes”**: Ángulos no obvios.
    - **C) 5 ideas “low effort”**: Rápidas de producir.
    - **D) 3 ideas “high impact”**: Más ambiciosas, más potentes.
3) **Filtrar**: Filtra y puntúa cada idea (1–5) en: Impacto, Claridad, Novedad, Esfuerzo, Viabilidad.
4) **Recomendar**: Devuelve un TOP 5 final.

## Checklist de Calidad
- [ ] ¿He concretado las ideas en lugar de usar términos genéricos?
- [ ] ¿Los hooks/títulos tienen tensión o curiosidad?
- [ ] ¿He incluido estructura y ejemplo para los formatos?
- [ ] ¿He mencionado dependencias inciertas y ofrecido alternativas?

## Reglas de calidad
- Nada de ideas genéricas (“mejorar tu productividad”). Concreta siempre.
- Si el usuario pide hooks/títulos, que sean cortos y con tensión/curiosidad.
- Si el usuario pide formatos, incluye: estructura + ejemplo de primer minuto.
- Si una idea depende de algo incierto, dilo y ofrece alternativa.

## Output (formato exacto)
Devuelve siempre:
1) **Preguntas rápidas**: (Solo si faltan datos).
2) **Ideas**: Con 4 bloques (A, B, C, D).
3) **TOP 5 recomendado**: Con puntuación y primer paso.
    - **Idea**: (1 línea)
    - **Por qué funciona**: (2 líneas)
    - **Primer paso**: (1 línea)
